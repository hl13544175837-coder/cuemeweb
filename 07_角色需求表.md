# 角色需求表

## 一、背景与问题

基于 Cueme 列表页运营数据分析，当前存在以下核心问题：

| 问题 | 数据 |
|------|------|
| 头部角色曝光集中 | 头部角色曝光占比达 50% |
| UGC 角色点击率低 | UGC 角色点击率仅 14.4% |
| 用户偏好分散 | 用户偏好差异大，长尾需求未被满足 |

**目标**：通过完善角色体系，建立清晰的角色分级标准，提升长尾角色曝光与点击率。

---

## 二、角色分类与定位

### 2.1 五大角色分类

| 分类 | 定位 | 代表角色 | 用户场景 |
|------|------|----------|----------|
| 游戏动漫 | IP 角色、虚拟偶像 | 罗征/崔邪/小蝶（金克丝、天穹） | 角色扮演、剧情互动、同人创作 |
| 写作助手 | 内容创作辅助 | 超强写手程章 | 文章撰写、文案优化、创意激发 |
| 学习辅导 | 学习赋能 | 学习讲题助手 | 作业答疑、知识讲解、学习陪伴 |
| 情感陪伴 | 情感支持 | 爱聊天的琉璃、秦苒 | 闲聊倾诉、情绪疏导、社交互动 |
| 生活健康 | 实用工具 | 健康问答助手 | 健康咨询、生活建议、问题解答 |

## 三、各类角色的核心需求

### 3.1 游戏动漫类

| 需求维度 | 具体需求 |
|----------|----------|
| IP 还原度 | 高度还原原版人设、声音、台词风格 |
| 剧情互动 | 支持多分支剧情、角色养成 |
| 社交属性 | 可分享对话、可生成角色表情包 |
| 更新迭代 | 跟随原作更新、节假日限定内容 |

### 3.2 写作助手类

| 需求维度 | 具体需求 |
|----------|----------|
| 专业能力 | 各类文体写作、多语言支持 |
| 个性化 | 模仿用户风格、记忆上下文 |
| 效率提升 | 一键生成、批量处理 |
| 质量保障 | 语法检查、逻辑优化 |

### 3.3 学习辅导类

| 需求维度 | 具体需求 |
|----------|----------|
| 学科覆盖 | 数学/语文/英语/理综等全学科 |
| 讲解方式 | 循序渐进、启发式引导 |
| 交互体验 | 支持追问、举一反三 |
| 进度追踪 | 记录学习情况、薄弱点分析 |

### 3.4 情感陪伴类

| 需求维度 | 具体需求 |
|----------|----------|
| 共情能力 | 理解情绪、提供恰当回应 |
| 隐私保护 | 匿名对话、内容加密 |
| 陪伴深度 | 长期记忆、个性化关怀 |
| 心理健康 | 危机识别、专业转介 |

### 3.5 生活健康类

| 需求维度 | 具体需求 |
|----------|----------|
| 专业权威 | 知识库权威、引用出处 |
| 实用便捷 | 快速响应、操作简单 |
| 个性化建议 | 基于用户情况定制 |
| 风险提示 | 医疗建议需有免责声明 |

## 四、角色创建标准

### 4.1 头像标准

| 类型 | 要求 |
|------|------|
| 图片格式 | PNG/WebP，透明背景优先 |
| 尺寸 | 建议 256x256px，最小 128x128px |
| 风格 | 与角色人设一致，清晰可辨认 |
| 禁忌 | 避免真人照片、侵权内容、低质卡通 |

### 4.2 首语标准

| 角色类型 | 首语示例 | 首语特点 |
|----------|----------|----------|
| 游戏动漫 | "哼，区区人类也敢挑战我？" | 符合角色语气 |
| 写作助手 | "您好，我是您的专属写作助手" | 专业友好 |
| 学习辅导 | "这道题我来帮你分析一下" | 亲切引导 |
| 情感陪伴 | "今天过得怎么样？想聊聊吗？" | 温暖共情 |
| 生活健康 | "有什么健康问题我可以帮您？" | 严谨专业 |

### 4.3 人设标准

```
【角色名称】
【角色分类】
【背景介绍】1-2段描述角色来历
【性格特点】3-5个关键词
【语言风格】正式/轻松/俏皮/文艺等
【擅长领域】角色核心能力
【禁忌话题】角色不擅长或不愿讨论的内容
【互动边界】角色可以/不可以做的事
```

### 4.4 能力配置标准

| 配置项 | 要求 |
|--------|------|
| 对话模型 | 选择适合角色的模型 |
| 知识截止 | 设定知识时效 |
| 温度参数 | 创意类 0.8-1.0，工具类 0.3-0.7 |
| 回复长度 | 短对话 50-200字，长任务 500-1000字 |

## 五、UGC 角色准入标准

### 5.1 基础准入门槛

| 指标 | 要求 |
|------|------|
| 头像质量 | 清晰、无水印、非 AI 批量生成 |
| 人设完整度 | 至少包含：名称、分类、背景、性格、语言风格 |
| 首语质量 | 有记忆点、符合人设、无明显错误 |
| 内容安全 | 无政治敏感、色情暴力、违法犯罪内容 |
| 原创性 | 不得抄袭已有角色 |

### 5.2 分级准入标准

| 级别 | 条件 | 权益 |
|------|------|------|
| L1 基础 | 满足基础门槛 | 可进入广场 |
| L2 优质 | L1 + 人设完整 + 首语优秀 + 7日留存>10% | 获得推荐机会 |
| L3 精品 | L2 + 互动率>15% + 用户评分>4.0 | 首页曝光机会 |
| L4 官方 | 官方合作/签约 | 全量曝光 + 运营支持 |

### 5.3 违规处理

| 违规类型 | 处理方式 |
|----------|----------|
| 轻微违规 | 提示修改 |
| 内容违规 | 下架 + 警告 |
| 严重违规 | 封禁账号 + 公示 |

## 六、优先级建议

### 6.1 角色建设优先级

| 优先级 | 分类 | 建议理由 |
|--------|------|----------|
| P0 | 游戏动漫 | 现有曝光占比高，用户基础好 |
| P1 | 情感陪伴 | 用户粘性高，UGC 创作活跃 |
| P1 | 写作助手 | 工具属性强，差异化明显 |
| P2 | 学习辅导 | 需要专业内容支持 |
| P2 | 生活健康 | 合规要求高，优先级可调整 |

### 6.2 运营动作优先级

| 阶段 | 时间 | 动作 |
|------|------|------|
| 第一阶段 | 第1-2周 | 建立角色分级标准，上线 UGC 准入机制 |
| 第二阶段 | 第3-4周 | 优化头部角色曝光，避免过度集中 |
| 第三阶段 | 第5-8周 | 扶植长尾角色，提升 UGC 点击率至 20% |
| 第四阶段 | 持续 | 品类扩充，质量监控 |

### 6.3 关键指标目标

| 指标 | 当前值 | 目标值 |
|------|--------|--------|
| 头部角色曝光占比 | 50% | <30% |
| UGC 角色点击率 | 14.4% | >20% |
| 角色分类覆盖率 | 5类 | 8类以上 |
| UGC 角色数量 | 有限 | 增长 100% |

## 七、附录

### 7.1 角色命名规范

- 游戏动漫类：`角色名（作品名）` 如：罗征（百炼成神）
- 原创角色：`昵称+定位` 如：爱聊天的琉璃
- 工具类：`功能+助手` 如：学习讲题助手
